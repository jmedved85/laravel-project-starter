# Laravel Project Starter

A Laravel project template with Docker setup for rapid development. No Laravel Sail dependency - pure Docker Compose for maximum flexibility and control.

## 🚀 Quick Start

### Prerequisites

- Docker & Docker Compose
- Git

### Initial Setup

Run the automated setup script:

```bash
chmod +x setup.sh
./setup.sh
```

This will:
1. Create `.env` from `.env.example`
2. Start Docker containers
3. Install Composer dependencies
4. Generate application key
5. Create database
6. Run migrations
7. Set proper permissions

**Application will be available at:**
- 🌐 Web: http://localhost:8080
- 🗄️ phpMyAdmin: http://localhost:8090
- ⚡ Vite dev server: http://localhost:5173

### Manual Setup (Alternative)

If you prefer manual setup:

```bash
# 1. Copy environment file
cp .env.example .env

# 2. Start containers
sh up.sh

# 3. Install dependencies
docker exec php-laravel_project_starter composer install

# 4. Generate application key
docker exec php-laravel_project_starter php artisan key:generate

# 5. Create database
docker exec mysql-laravel_project_starter mysql -uroot -ppass1234 -e "CREATE DATABASE IF NOT EXISTS \`laravel-project-starter_dev\` CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 6. Run migrations
docker exec php-laravel_project_starter php artisan migrate

# 7. Fix permissions
docker exec php-laravel_project_starter chown -R www-data:www-data /var/www/html/storage /var/www/html/bootstrap/cache
docker exec php-laravel_project_starter chmod -R 775 /var/www/html/storage /var/www/html/bootstrap/cache
```

## 🐳 Docker Services

This project includes the following services:

- **php** - PHP 8.4 FPM (Alpine) with common extensions
- **nginx** - Nginx web server
- **mysql** - MySQL 8.0.39
- **phpmyadmin** - Database management UI
- **node** - Node.js 24 for frontend assets

## 📋 Common Commands

### Container Management

```bash
sh up.sh                  # Start all containers
sh down.sh                # Stop all containers
docker compose logs -f    # View logs
```

### Development

```bash
# Artisan commands
docker exec php-laravel_project_starter php artisan migrate
docker exec php-laravel_project_starter php artisan migrate:fresh --seed
docker exec php-laravel_project_starter php artisan tinker

# Composer
docker exec php-laravel_project_starter composer install
docker exec php-laravel_project_starter composer update
docker exec php-laravel_project_starter composer require vendor/package

# Testing
sh test.sh
# or
docker exec php-laravel_project_starter php artisan test

# Code quality
sh pint.sh                                      # Fix code style
docker exec php-laravel_project_starter ./vendor/bin/phpstan analyse  # Static analysis

# Cache management
docker exec php-laravel_project_starter php artisan config:clear
docker exec php-laravel_project_starter php artisan cache:clear
docker exec php-laravel_project_starter php artisan view:clear

# Frontend assets
docker compose run --rm node npm install
docker compose run --rm node npm run dev
docker compose run --rm node npm run build
```

### Database

```bash
# Access MySQL CLI
docker exec -it mysql-laravel_project_starter mysql -uroot -ppass1234

# Database backup
docker exec mysql-laravel_project_starter mysqldump -uroot -ppass1234 laravel-project-starter_dev > backup.sql

# Database restore
docker exec -i mysql-laravel_project_starter mysql -uroot -ppass1234 laravel-project-starter_dev < backup.sql
```

### Shell Access

```bash
# PHP container
docker exec -it php-laravel_project_starter sh

# Node container
docker compose run --rm node sh

# MySQL container
docker exec -it mysql-laravel_project_starter sh
```

## 🔧 Configuration

### Environment Variables

Key configuration in `.env`:

```env
APP_NAME=Laravel
APP_ENV=local
APP_KEY=base64:...                    # Generated by setup.sh
APP_URL=http://localhost:8080

DB_CONNECTION=mysql
DB_HOST=mysql                         # Docker service name
DB_PORT=3306
DB_DATABASE=laravel-project-starter_dev
DB_USERNAME=root
DB_PASSWORD=pass1234

VITE_DEV_SERVER_URL=http://localhost:5173
```

### Ports

- **8080** - Nginx (web application)
- **8090** - phpMyAdmin
- **5173** - Vite dev server
- **3308** - MySQL (if you need external access)

## 🛠️ Troubleshooting

### Common Issues

**500 Internal Server Error**
```bash
# Check logs
docker logs php-laravel_project_starter
docker logs nginx-laravel_project_starter

# Fix permissions
docker exec php-laravel_project_starter chown -R www-data:www-data /var/www/html/storage /var/www/html/bootstrap/cache
docker exec php-laravel_project_starter chmod -R 775 /var/www/html/storage /var/www/html/bootstrap/cache
```

**Database connection issues**
```bash
# Check if MySQL is running
docker ps | grep mysql

# Wait for MySQL to be ready
docker exec mysql-laravel_project_starter mysqladmin ping -h localhost -uroot -ppass1234
```

**"No application encryption key"**
```bash
docker exec php-laravel_project_starter php artisan key:generate
```

## 📦 Project Structure

```
project/
├── app/                    # Application code
├── bootstrap/              # Framework bootstrap files
├── config/                 # Configuration files
├── database/              # Migrations, seeders, factories
├── docker/                # Docker configuration
│   ├── nginx/            # Nginx Dockerfile & config
│   └── php/              # PHP Dockerfile & config
├── public/                # Public web root
├── resources/             # Views, CSS, JS
├── routes/                # Route definitions
├── storage/               # Logs, cache, uploads
├── tests/                 # Tests
├── docker-compose.yml     # Docker services definition
├── setup.sh              # Initial setup automation
├── up.sh / down.sh       # Container management
└── .env                  # Environment variables
```

## About Laravel

Laravel is a web application framework with expressive, elegant syntax. We believe development must be an enjoyable and creative experience to be truly fulfilling. Laravel takes the pain out of development by easing common tasks used in many web projects, such as:

- [Simple, fast routing engine](https://laravel.com/docs/routing).
- [Powerful dependency injection container](https://laravel.com/docs/container).
- Multiple back-ends for [session](https://laravel.com/docs/session) and [cache](https://laravel.com/docs/cache) storage.
- Expressive, intuitive [database ORM](https://laravel.com/docs/eloquent).
- Database agnostic [schema migrations](https://laravel.com/docs/migrations).
- [Robust background job processing](https://laravel.com/docs/queues).
- [Real-time event broadcasting](https://laravel.com/docs/broadcasting).

Laravel is accessible, powerful, and provides tools required for large, robust applications.
